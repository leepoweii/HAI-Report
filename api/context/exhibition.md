歡迎來到第四組的報告展覽！本展覽結合人工智慧與虛擬導覽，打造沉浸式藝術體驗。

## AI 工具展示區

本展覽介紹三款 AI 創作工具，分別展示於二樓牆面的影片中。

### Character AI

#### 什麼是 Character AI
Character AI 是一個讓使用者與各種 AI 人格（角色）進行長篇、多輪對話的平台。你可以創建自己的角色，設定性格、指令、示範對話與記憶，並公開分享給其他人使用。它偏重「擬人化、故事性與角色扮演」的聊天體驗，而非工具整合或任務自動化。

#### 核心功能
- **多輪對話**: 支援長篇、連貫的對話體驗
- **角色創建**: 自訂角色性格、口吻與世界觀
- **群組聊天 (Rooms)**: 多個角色同時對話，開啟創意互動
- **Character Voice**: AI 角色可朗讀回覆內容
- **Character Calls**: 雙向語音通話功能

#### 2025 重要更新
- **Chat Memory**: 為每個角色提供 400 字記憶空間，記住重要對話內容
- **Image Attachments**: 強化圖片識別能力，角色可理解並回應上傳的照片
- **Scenes**: 創建自訂場景，設定對話情境與背景故事
- **Model Picker**: 選擇不同 AI 模型，找到最適合的對話風格
- **多語言支援**: 「Pawly」實驗性聊天風格，強化多語言能力

#### 使用場景
- 創意寫作與故事共創
- 語言學習與對話練習
- 角色扮演與沉浸式娛樂
- 教育互動與知識探索

### Viggle AI

#### 什麼是 Viggle AI
Viggle AI 是一款以範例驅動的影像生成與角色替換工具。只需一張人物照片，即可套用現成的動作模板與場景，快速生成帶有口型同步與肢體動作的短影片。它因「一鍵做梗圖」的流暢體驗，在社群上非常流行，全球用戶已超過 400 萬人。

#### 核心技術
Viggle 採用自研的 **JST-1 video-3D foundation model**，這是一個理解物理規律的影片生成模型，能模擬真實的肢體動作與物理效果。同時也整合 Google 的 Gemini 2.0 Flash 與 Imagen 3 技術。

#### 核心功能
- **Mix**: 單張照片替換影片中的角色，保留原始動作與鏡位
- **Multi**: 讓多位角色同時出現在同一支影片模板中
- **Move**: 套用動作模板（跳舞、跑步、耍帥等），快速生成短片
- **Animate**: 使用參考影片讓靜態圖片動起來
- **Mic**: 根據音訊做口型與表情同步，適合對嘴或說唱段落
- **Rap**: 針對音樂節奏與歌詞做畫面節奏與口型匹配

#### 2025 新功能
- **Live Mode**: 即時變身為任何角色，進行直播互動
- **Motion Capture**: 無需專業攝影棚即可進行動作捕捉，內建綠幕支援
- **VFX Tools**: 輕鬆製作 CGI 角色、超級英雄或神話生物的特效
- **Meme Maker**: 超過 1000+ 梗圖模板，快速製作社群爆款內容

#### 技術規格
- **輸入格式**: JPG、PNG 圖片；MP4 影片
- **輸出品質**: 最高 1080p（付費方案無浮水印）
- **影片長度**: 一般 5-10 秒，進階模式可更長
- **版權**: 生成內容免版稅，可用於商業用途

#### 定價方案
- **免費版**: 每日 5 支影片（relaxed 模式）、2 個同時生成、15 天儲存
- **付費版**: 起價 $9.99/月（年繳 8 折）

#### 使用場景
- 社群梗圖、對嘴影片製作
- Reels / TikTok 短影音創作
- 角色分鏡與動作概念打樣
- 行銷素材與廣告製作

### Mootion AI

#### 什麼是 Mootion AI
Mootion 是一款用 AI 一鍵生成分鏡、腳本與影片的創作平台。它能從文字、音訊、連結等多種輸入自動「規劃故事、分鏡、配音與視覺」，並可即時編修與下載成片。特別適合教育場景，教師與學生可用課文或改寫文本生成動畫。

#### 核心功能
- **自動分鏡生成**: 輸入一句話或完整腳本，自動產生場景分解
- **多元輸入**: 支援文字提示、音訊檔案、網頁連結等輸入方式
- **AI Avatar**: 擬真人的 AI 虛擬角色，可作為影片主角
- **多語言輸出**: 支援包含中文在內的多國語言
- **即時編輯**: 生成後可微調分鏡、配音與視覺效果

#### 2025 重要更新
- **角色一致性引擎**: 重建風格引擎，確保角色在不同場景中維持一致外觀（服裝、長相、整體風格）
- **新圖像生成模型**: 優化角色與場景的視覺品質
- **遊戲分鏡功能**: 專為遊戲開發設計，理解遊戲機制、角色互動與鏡頭構圖

#### 風格選項
支援 12 種視覺風格：
- Retro（復古）
- Comics（漫畫）
- Photorealistic（擬真）
- 3D（立體）
- Cartoon（卡通）
- Anime（動漫）
- Painting（繪畫）
- Clay Art（黏土）
- Faded Film（復古膠片）
- Water Ink（水墨）
- Line Art（線稿）
- Pixel Art（像素）

#### 效能表現
- 3 分鐘分鏡序列僅需約 2 分鐘生成
- 比業界平均速度快 65%

#### 輸出格式
- 分鏡圖
- 縮圖
- MP4 影片（最多 16 個鏡頭）

#### 使用場景
- 教育動畫製作（課文視覺化）
- 影片企劃與分鏡打樣
- 兒童故事動畫
- 行銷影片快速產出

## 虛擬世界導覽

### 空間配置
- **一樓大廳**: 進入展間的起點，可與其他參觀者互動交流
- **二樓展區**: 牆面上展示 AI 工具介紹影片，目前共有 3 支影片

### 互動方式
- 使用鍵盤或觸控移動探索空間
- 點擊牆面影片觀看內容
- 透過右側聊天面板與 AI 導覽員對話

### 目前限制
由於 VIVERSE Persona 需另外申請使用，目前以自建 Chatbot 替代：
- 僅支援文字輸入，無法語音對話
- Chatbot 回答內容有限，無法涵蓋所有展覽資訊
- Chatbot 無法與 VIVERSE 3D 世界互動（無法幫使用者播放影片、移動位置或控制任何展間功能）
- Chatbot 無法即時感知使用者在展間的位置或正在觀看的影片

## 技術實作

### 系統架構
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   使用者    │ ──▶ │  網頁前端   │ ──▶ │  後端 API   │
│  (瀏覽器)   │     │  (Nuxt 4)   │     │  (FastAPI)  │
└─────────────┘     └──────┬──────┘     └──────┬──────┘
                          │                    │
                    ┌─────▼─────┐        ┌─────▼─────┐
                    │  VIVERSE  │        │  OpenAI   │
                    │  3D 世界  │        │    API    │
                    └───────────┘        └───────────┘
```

### 核心技術

| 層級 | 技術 | 說明 |
|-----|------|-----|
| 前端框架 | Nuxt 4 + Vue 3 | 現代化網頁應用框架，SPA 模式運行 |
| 樣式設計 | Tailwind CSS | 深色太空主題，響應式設計 |
| 3D 世界 | VIVERSE Worlds | HTC 虛擬實境平台，透過 iframe 嵌入 |
| 後端框架 | FastAPI | Python 高效能非同步 API 框架 |
| AI 模型 | OpenAI Responses API | 支援串流對話與網路搜尋功能 |
| 部署平台 | Zeabur + Docker | 雲端容器化部署，自動化 CI/CD |

### 特色功能
- **即時串流**: 對話內容逐字顯示，無需等待完整回覆
- **雙語支援**: 自動偵測並匹配使用者語言（中文/英文）
- **動態知識庫**: 展覽內容存於獨立檔案，可隨時更新無需修改程式碼
- **LINE Bot 整合**: 可透過 LINE 與導覽員互動（選配功能）

## 常見問題

**Q: 如何與 AI 導覽員互動？**
A: 在網頁右側的聊天面板中輸入文字訊息，AI 導覽員會即時回覆。你可以詢問關於展覽內容、AI 工具介紹，或任何你感興趣的問題。

**Q: 支援哪些語言？**
A: 導覽員支援中文與英文，會自動偵測並以相同語言回覆。

**Q: 如何探索 3D 虛擬世界？**
A: 使用鍵盤方向鍵或 WASD 移動，滑鼠控制視角。點擊牆面上的影片可以觀看 AI 工具介紹。注意：AI 導覽員無法幫你播放影片或控制展間，請自行在 3D 世界中操作。

**Q: 可以在手機上使用嗎？**
A: 可以。網頁採用響應式設計，手機上會自動調整為上下分割畫面（VIVERSE 世界在上、聊天面板在下）。

**Q: AI 導覽員的知識來源是什麼？**
A: 導覽員的知識來自本展覽的知識庫，並可透過網路搜尋補充最新資訊。對於展覽範圍外的問題，會誠實告知知識限制。

**Q: 如何回報問題或提供建議？**
A: 請聯繫團隊成員或透過課程管道回報。我們歡迎任何改善建議！
